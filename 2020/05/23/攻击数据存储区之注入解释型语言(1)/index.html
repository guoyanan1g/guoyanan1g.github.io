攻击数据存储区之注入解释型语言(1)

一、SQL注入

1、原理

解释型语言的执行方式产生了_代码注入_的漏洞。利用这种漏洞提交含语法的句式，从而对程序进行攻击

2、基本漏洞

1. 假设有一个登陆界面，需提交用户名及密码。若已知用户名为guoyanan，则可构造guoyanan' --输入进用户名,程序将执行SELECT * FROM XX WHERE username='guoyanan' -- '.........,'--',--后面提交的密码会被认为是注释而被忽略，从而起到绕过登录的作用。

*2.OR 1=1 -- 一样可以起到这种作用，提交后，程序将执行SELECT * FROM XX WHERE username=' ' OR 1=1 --,执行的结果将会返回XX表中的所有数据 *

3.因为提交的句式的末尾会生成一个 ‘ ，我们也可以提交guoyanan' OR 'a'='a从而不用注释也可以平衡执行语句的引号。

3.INSERT语句

INSERT用于创建一新行，例如INSERT INTO XX (name,year,sex,isadmin) VALUE ('guoyanan',19,1,0)，其中isadmin代表是否具有管理员权限。

我们可以建立一个新行，提交admin',100,1,1) --来创建一个具有管理员权限的帐号，若是字段中含有密码password,我们就可以随便设立一个密码，通过已知密码的管理员帐号进入数据库了。

另外，通过拦截到的数据库的版本字符串，把它插入自己用户资料的一个字段中；一般数据库会显示其版本信息。

有时候我们并不知道有多少个字段，我们就可以逐个增加，从guoyanan',1) --到guoyanan',1,1,1....) --,因为大部分数据库会自动将整数转为字符串。若是1被拒绝，可以尝试200，2000之类。

4、UPDATE语句

UPDATE XX SET password ='new-pass' WHERE user='xx' and password='old-pass',将某一行原有某字段的值替换为新的。我们依然可以通过绕过来更新别人的密码。例如输入XX' --，如果将注释符替换为or 1=1 --,那就能替换所有用户的密码了，

5、DELETE语句

DELETE用于删除表中一个或多个行。与UPDATE类似，可以恶意破坏数据
